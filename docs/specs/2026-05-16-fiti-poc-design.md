# fiti POC — Native Swift Telestrator (Design)

Date: 2026-05-16
Status: Implemented. Deltas captured in [`2026-05-16-fiti-hardening-plan.md`](./2026-05-16-fiti-hardening-plan.md) — notably the global `Cmd+Opt+Z` hotkey now works via an accessibility-permission check, contradicting two statements below (the out-of-scope bullet and the `NSEventInputSource` description). Stack also bumped to Swift 6 language mode.

## What this is

A proof-of-concept native macOS port of [telestrator](https://github.com/steveruizok/telestrator) (the MIT-licensed Electron drawing-overlay app vendored at `.llm/telestrator/`). The POC validates three concepts before we invest further:

1. A borderless, transparent, always-on-top macOS window with cursor click-through when inactive.
2. A hexagonal Core ↔ adapters split that keeps domain logic in pure Swift and pushes AppKit / Core Graphics / Network code to thin adapters tested in isolation.
3. A dev-mode HTTP introspection-and-control surface so Claude Code (and shell scripts) can observe and drive the running app without needing screen / pointer access.

This is intentionally narrow. Shapes, fading, pen pressure, perfect-freehand-quality strokes, a toolbar, keyboard shortcuts beyond bare activate/deactivate, multi-display handling, codesigning, and CI matrices are all deferred. The POC is the smallest thing that lets us know whether the architecture and dev loop work.

## Out of scope (deliberate punts)

- Rect / ellipse / arrow / eraser-as-tool. (Eraser-by-id is in the *data model* from day one because it falls out for free; the UI never invokes it.)
- Mark fading.
- Pressure-sensitive variable-width strokes / perfect-freehand port. POC uses uniform CGPath strokes.
- Pen and touch input — pointer events come from the mouse only.
- A toolbar, color picker, size selector. One hardcoded color, one hardcoded width.
- Keyboard shortcuts beyond `Cmd+Opt+Z` (activate), `Esc` (deactivate), and `Cmd+K` (clear). No undo/redo on the keyboard for POC — those go through HTTP.
- Global activation shortcut via accessibility permissions. POC's `Cmd+Opt+Z` only works when the window has focus. Same caveat for `Cmd+K`.
- `.app` bundling for distribution beyond development. xcodegen produces an .app, but we don't sign, notarize, or ship it.
- Multi-display awareness. POC targets the main display.
- Automerge / CRDT sync. The `FitiDoc` shape is CRDT-shaped so we can swap an Automerge impl in later without changing the `Editor` or HTTP API surface, but the POC's backing store is a plain Swift struct.

## Why not just clone telestrator's design

Telestrator's drawing model is `marks: CompleteMark[]` plus `redos: CompleteMark[]` — two parallel arrays of full stroke objects, plus a `fading` array. Undo is `pop one off marks, push it onto redos`. That works for a small Electron app but doesn't survive forward changes well:

- No stable stroke identity, so "erase a specific stroke", "recolor an existing stroke", or "drag/scale a stroke" don't have a referent.
- Eraser is `globalCompositeOperation = "destination-out"` painted over existing pixels. That conflates a behavioral concept (delete a stroke) with a rendering technique (subtract pixels), and breaks any future serialization, sync, or hit-testing story.
- Undo can only undo "last completed mark"; adding any new mutation kind requires bolting on a parallel history mechanism.

The collaborative-drawing research spike at `../scratch/scratch` solved the same problem differently. Its data model is `strokes: Map<StrokeId, Stroke>` plus `strokeOrder: StrokeId[]`. Geometry freezes at `endStroke`; later edits target a per-stroke `transform: {x, y, scale, rotate}` field. Eraser deletes the map entry. Undo is an `InverseOp` stack — small data records describing how to reverse a change — applied as forward edits, not as history rewinds.

fiti adopts the scratch pattern because:
- It works whether the backing store is plain Swift, Automerge, or anything in between. The `Editor` surface is the same.
- Stroke identity gives the HTTP API stable references (`GET /strokes/{id}`).
- New mutation kinds (color, width, transform) add one `InverseOp` case each. No new history machinery.
- Eraser semantics are unambiguous and serialization-clean from day one.

### Hexagonal boundary enforcement

xcodegen's `project.yml` declares two targets:

| Target | Type | sources |
|---|---|---|
| `fiti` | `application` | `Sources` |
| `fiti-unit` | `bundle.unit-test` | `Tests`, `Sources/Core`, `Sources/DevHTTP` |

The test target's `sources` list is the load-bearing line. `Sources/AppKit` and `Sources/App` are not compiled into the test bundle at all, so `CoreTests` cannot import AppKit even if a future contributor tries to. This mirrors montty's `montty-unit` target which only compiles `Sources/Model` + `Sources/Persistence` into the test bundle.

The dependency rule on top of that: nothing under `Sources/Core` may `import AppKit`, `import CoreGraphics`, or `import Network`. CI enforces this via a grep step in `just check`.

## Domain model (`Sources/Core`)

```swift
public typealias StrokeId = String
public typealias TimeInterval = Double

public enum PointerType: String, Equatable, Codable {
    case mouse, pen, touch
}

public struct RGBA: Equatable, Codable {
    public var r, g, b, a: Double   // sRGB, 0...1
}

public struct StrokePoint: Equatable, Codable {
    public var x: Double
    public var y: Double
    public var pressure: Double     // 0...1; 0.5 for mouse input in POC
}

public struct Transform: Equatable, Codable {
    public var x, y: Double
    public var scale: Double
    public var rotate: Double       // degrees
    public static let identity = Transform(x: 0, y: 0, scale: 1, rotate: 0)
}

public struct Stroke: Equatable, Codable {
    public let id: StrokeId
    public var color: RGBA
    public var width: Double
    public var transform: Transform     // identity in POC; set up for drag/scale later
    public var points: [StrokePoint]    // append-only while drawing; frozen at endStroke
    public let pointerType: PointerType
    public let pressureEnabled: Bool
    public let createdAt: TimeInterval
}

public struct FitiDoc: Equatable, Codable {
    public var strokes: [StrokeId: Stroke]
    public var strokeOrder: [StrokeId]
    public static let empty = FitiDoc(strokes: [:], strokeOrder: [])
}

public enum InverseOp: Equatable {
    case deleteStroke(StrokeId)
    case restoreStroke(snapshot: Stroke, atIndex: Int)
    case deleteStrokes([StrokeId])
    case restoreStrokes(entries: [StrokeRestoreEntry])
    // setColor / setWidth / setTransform added when those edits land
}

public struct StrokeRestoreEntry: Equatable {
    public let snapshot: Stroke
    public let atIndex: Int
}
```

`Stroke.pointerType` and `Stroke.pressureEnabled` exist now so a future perfect-freehand-port renderer can decide between honoring pressure and simulating it — exactly the distinction telestrator's `simulatePressure` flag draws. POC writes `(mouse, false, 0.5)` for every point.

## Editor (`Sources/Core/Editor`)

`Editor` is the sole mutation surface. It owns the document and the undo/redo stacks. All mutations go through method calls; ports for `Clock` and `IdGenerator` keep tests deterministic.

```swift
public protocol Clock { func now() -> TimeInterval }
public protocol IdGenerator { func newStrokeId() -> StrokeId }

public final class Editor {
    public private(set) var doc: FitiDoc = .empty
    public private(set) var undoStack: [InverseOp] = []
    public private(set) var redoStack: [InverseOp] = []
    public private(set) var currentStrokeId: StrokeId?

    private let clock: Clock
    private let ids: IdGenerator

    public init(clock: Clock, ids: IdGenerator)

    public func startStroke(color: RGBA, width: Double, pointerType: PointerType) -> StrokeId
    public func appendPoint(_ p: StrokePoint)
    public func endStroke()

    @discardableResult public func eraseStroke(_ id: StrokeId) -> Bool
    public func clear()

    @discardableResult public func undo() -> Bool
    @discardableResult public func redo() -> Bool

    public func subscribe(_ listener: @escaping (ChangeKind) -> Void) -> Cancellable
}

public enum ChangeKind { case local, remote }

/// Returned by `Editor.subscribe`. Call to unsubscribe. Kept as a closure so Core
/// doesn't import Combine — Core depends only on Swift stdlib + Foundation.
public typealias Cancellable = () -> Void
```

Invariants:

- `pushUndo(op)` always clears `redoStack`. Any new edit invalidates redos. Direct port of scratch's pattern; mirrors what most editors do.
- `undo()` pops from `undoStack`, applies the inverse, and pushes the inverse-of-the-inverse onto `redoStack`. `redo()` is symmetric.
- `eraseStroke` and `clear` *are* part of the data model in POC even though the UI doesn't expose them. They get exercised via HTTP routes (`POST /clear`, `POST /strokes/{id}/erase`) and by tests.
- All mutations call `listener({source: .local})` so adapters can repaint.

## Control (`Sources/Core/Control`)

`AppController` is the activation state machine. It bridges raw input events (from `AppKit` or `DevHTTP`) to `Editor` calls and owns the click-through toggle.

```swift
public final class AppController {
    public enum Mode: Equatable {
        case inactive          // click-through on; cursor passes through window
        case activeIdle        // cursor captured; not currently drawing
        case activeDrawing     // pointer is down, building a stroke
    }

    public private(set) var mode: Mode = .inactive
    public let editor: Editor

    public init(editor: Editor, window: WindowControl)

    public func activate()
    public func deactivate()

    public func pointerDown(_ p: StrokePoint)
    public func pointerMoved(_ p: StrokePoint)
    public func pointerUp()

    public func clear()              // pass-through to editor.clear()
}
```

Behavior:

- `activate()` → `mode = .activeIdle`, `window.setClickThrough(false)`, `window.focus()`.
- `deactivate()` → if `.activeDrawing` first call `editor.endStroke()`, then `mode = .inactive`, `window.setClickThrough(true)`.
- `pointerDown` while `.activeIdle`: `editor.startStroke(...)`, `editor.appendPoint(p)`, `mode = .activeDrawing`.
- `pointerMoved` while `.activeDrawing`: `editor.appendPoint(p)`.
- `pointerUp` while `.activeDrawing`: `editor.endStroke()`, `mode = .activeIdle`.

**HTTP routes call `AppController` methods directly and bypass the activation gate** — this matches scratch's "the CLI drives the same `Editor` the browser does" principle and keeps Claude-driven tests scriptable without `POST /activate` boilerplate.

    public var height: Double

## Adapters

### `Sources/AppKit`

- **`TransparentWindow: NSWindow`** — borderless, `level = .floating`, `isOpaque = false`, `backgroundColor = .clear`, occupies main screen. Conforms to `WindowControl`: `setClickThrough` toggles `ignoresMouseEvents`; `focus()` calls `makeKeyAndOrderFront(nil)`.
- **`CanvasView: NSView`** — owns two layers (committed marks, in-progress) and implements `Renderer`. Each `render(frame)` clears the in-progress layer and redraws the current stroke; if `strokes` differs from last frame it redraws the committed layer too (only on stroke completion / undo / redo / clear, so the committed layer is cheap).
- **`NSEventInputSource: InputSource`** — overrides `mouseDown` / `mouseDragged` / `mouseUp` on `CanvasView`; uses `NSEvent.addLocalMonitorForEvents` for `Cmd+Opt+Z` (activate), `Esc` (deactivate), and `Cmd+K` (clear). No global monitor → no accessibility permission required for POC. The local monitor only fires when our app is key-focused; `Cmd+K` therefore requires the overlay to be active (or at least frontmost), unlike `Cmd+Opt+Z` which is the entry point.

### `Sources/DevHTTP`

Hand-rolled `NWListener`-based HTTP responder, mirroring the pattern used in `../montty/Sources/App/DebugServer.swift` (158 lines, same shape) and `../limn/packages/desktop/Limn/DebugServer.swift` (305 lines, similar). Zero external Swift package dependencies.

The server binds to `127.0.0.1:9876` only — matching the port both montty and limn use, so Claude's existing muscle memory carries over. Started only when `fiti --dev` is passed.

### HTTP routes

| Method | Path | Behavior |
|---|---|---|
| GET  | `/state`              | JSON: `{ mode, clickThrough, canvasSize, currentStrokeId, undoDepth, redoDepth }` |
| GET  | `/doc`                | JSON: full `FitiDoc` |
| GET  | `/strokes/{id}`       | JSON: single stroke (404 if absent) |
| GET  | `/snapshot.png`       | renders current frame to a headless `CGContext`, returns PNG. Same call path the `CanvasView` renderer uses. |
| POST | `/activate`           | `controller.activate()` |
| POST | `/deactivate`         | `controller.deactivate()` |
| POST | `/pointer`            | body `{ event: "down\|move\|up", x: N, y: N, pressure?: N }` → `controller.pointerDown/Moved/Up` |
| POST | `/strokes/{id}/erase` | `editor.eraseStroke(id)` |
| POST | `/clear`              | `editor.clear()` |
| POST | `/undo`               | `editor.undo()` |
| POST | `/redo`               | `editor.redo()` |

Responses are `application/json` for everything except `/snapshot.png`. Non-2xx responses include `{ "error": "..." }`. The server holds a strong reference to the `AppController` (handed in at construction); it never touches `AppKit` types.

## Wiring (`Sources/App`)

`main.swift` parses argv (`--dev`, `--port`, `--log`), constructs the dependency graph, then runs `NSApplication.shared.run()`.

```swift
// Concrete adapter impls live next to their port-using code:
//   Sources/App:    SystemClock, ULIDGenerator
//   Sources/AppKit: TransparentWindow, CanvasView, NSEventInputSource
// Test doubles (VirtualClock, SeededIdGenerator, FakeRenderer, RecordingWindow)
// live in Tests/CoreTests and Tests/DevHTTPTests.

let clock      = SystemClock()
let ids        = ULIDGenerator()
let editor     = Editor(clock: clock, ids: ids)
let window     = TransparentWindow()
let canvas     = CanvasView(frame: window.contentRect)
let input      = NSEventInputSource(view: canvas)
let controller = AppController(editor: editor, window: window)

input.onPointerDown   = { controller.pointerDown($0) }
input.onPointerMoved  = { controller.pointerMoved($0) }
input.onPointerUp     = { controller.pointerUp() }
input.onActivate      = { controller.activate() }
input.onDeactivate    = { controller.deactivate() }

// On every doc change, build a fresh RenderFrame from editor state and hand
// it to the canvas. RenderFrame.from(editor:) is a static helper in Core.
let _ = editor.subscribe { _ in canvas.render(RenderFrame.from(editor: editor)) }

let devServer = args.dev
    ? try DevHTTPServer(controller: controller, editor: editor, port: args.port).start()
    : nil

NSApplication.shared.run()
```

`Sources/App` is the only place `import AppKit` and `import Network` and the concrete `Editor`/`AppController` coexist. Nothing else does cross-layer wiring.

## Testing strategy

Swift Testing (`import Testing`), per montty/limn convention. Target the full suite to run in under 5 seconds.

### `CoreTests`
- Drive `Editor` with scripted method calls, assert `doc`, `undoStack`, `redoStack`, `currentStrokeId` after each. Pure, no AppKit, sub-millisecond.
- Coverage targets: `startStroke → appendPoint × N → endStroke → undo → assert doc empty → redo → assert doc matches snapshot`; `eraseStroke → undo → assert restored at correct strokeOrder index`; `clear → undo → assert all restored`; `undo with empty stack → returns false`; `pushUndo clears redos`.
- Drive `AppController` against a fake `WindowControl` that records `setClickThrough` calls. Assert mode transitions and click-through side effects.

### `DevHTTPTests`
- Construct a fake `AppController` exposing the same method surface. Start `DevHTTPServer` on an ephemeral port. Hit each route with `URLSession`.
- Assert response shapes for `/state`, `/doc`, `/strokes/{id}`.
- For mutation routes, assert the matching method ran on the fake.
- For `/snapshot.png`, hit a `FakeRenderer` that produces a deterministic 1x1 PNG; assert content-type and bytes.

### Manual end-to-end (no automated runner in POC)
- `just dev`, then run `curl` and `just inspect-*` recipes by hand. The dev surface itself is the end-to-end harness — adding XCUITest for window interactions is more setup than the POC earns.

## Behavioral acceptance criteria

The POC is done when, with `just dev` running and no human input:

1. `curl -X POST localhost:9876/activate` returns 200; subsequent `GET /state` reports `mode: "activeIdle"`, `clickThrough: false`.
2. A scripted sequence of `POST /pointer` (down/move×N/up) results in `GET /doc` returning a `FitiDoc` with one stroke, N+1 points, identity transform, configured color and width.
3. `GET /snapshot.png` renders an image where the stroke is visible at the correct pixel coordinates.
4. `POST /undo` empties the doc; `POST /redo` restores it byte-identically.
5. `POST /clear` followed by `POST /undo` restores all strokes at their original `strokeOrder` positions.
6. `POST /deactivate` returns 200; `GET /state` reports `clickThrough: true`. Real-mouse interaction passes through the window to the desktop beneath.
7. `just check` passes (tests + lint + build).

## Open questions to settle during implementation

- HiDPI: `RenderFrame.canvasSize` is logical points. `CanvasView` reads `window.backingScaleFactor` and scales the off-screen CGContext for `/snapshot.png` accordingly. Snapshot output is at logical resolution × backing scale.
- Color space: strokes carry sRGB RGBA. `CGRenderer` uses `CGColorSpace(name: CGColorSpace.sRGB)`.
- ULID vs UUID for `StrokeId`: ULID gives sortable ids without an explicit `createdAt` join. Tests use a `SeededIdGenerator(seed:)` for determinism.
- `LSUIElement = true` in Info.plist so the app doesn't show a Dock icon — appropriate for an always-on-top overlay.

## Decision log

- **2026-05-16 — Adopt scratch-style data model over telestrator's two-stack approach.** Strokes get stable identity (`StrokeId`), geometry freezes at `endStroke`, eraser deletes by id (not by `destination-out` painting), undo/redo via `InverseOp` records applied as forward edits. Costs ~30 extra LOC over the naive two-stack approach. Earns: stable HTTP references, identity-aware future features (drag/resize/recolor), CRDT-shape (Automerge can swap in later without changing `Editor`), clean serialization.
- **2026-05-16 — xcodegen + xcodebuild, not pure SwiftPM.** Matches montty / limn convention. Test target's `sources:` list enforces hexagonal boundary at the build-graph level. Loses fast `swift test`; gains consistent build / IDE / .app-bundling story.
- **2026-05-16 — NWListener for dev HTTP, no FlyingFox.** Both montty (158 LOC) and limn (305 LOC) hand-rolled NWListener-based debug servers. Zero external Swift package dependencies. Pattern is proven twice.
- **2026-05-16 — Dev HTTP bypasses activation gate.** HTTP routes call `AppController` methods directly without checking `mode`. Matches scratch's "CLI drives the same Editor as the browser" stance; makes Claude scripts simpler.
- **2026-05-16 — Port 9876 for dev HTTP.** Same as montty and limn. Carries existing muscle memory.
- **2026-05-16 — Punt Automerge.** `FitiDoc` shape is CRDT-shaped (keyed map + ordered list, identity-bearing strokes, frozen points + transform field) so the migration is purely internal to `Editor`. POC remains scope-focused.
- **2026-05-16 — Punt perfect-freehand port.** POC uses uniform CGPath strokes. `Stroke.pressureEnabled` and `Stroke.pointerType` exist in the model so a future renderer can switch on them without restructuring.
- **2026-05-16 — Eraser-by-id and undo/redo are in the data model from day one even though the POC UI doesn't expose them.** Exercised via HTTP routes and tests. Avoids restructuring when the toolbar arrives.
- **2026-05-16 — `Cmd+K` clears the screen.** Matches terminal muscle memory (iTerm / Terminal.app). Three keyboard shortcuts in POC: `Cmd+Opt+Z` (activate), `Esc` (deactivate), `Cmd+K` (clear). Undo / redo / erase stay HTTP-only. The `Cmd+K` local monitor only fires when our app is key-focused, so clearing from click-through mode requires `Cmd+Opt+Z` first.
- **2026-05-16 — UUID-based stroke ids instead of ULIDs.** Spec called for `ULIDGenerator` for sortable stable ids. Implementation uses `UUIDStrokeIds` (Foundation's `UUID().uuidString`) instead. Sortability isn't load-bearing because `FitiDoc.strokeOrder` is the canonical ordering; uniqueness is the only property the model relies on. Zero new dependencies, no algorithm to maintain. The id-format change is internal and can be revisited if any feature ever cares about timestamp-from-id.
- **2026-05-16 — DevHTTP handlers hop to main queue.** `DevHTTPServer.readRequest` runs on a private `DispatchQueue`. Route handlers touch `AppController` / `Editor` / AppKit, which are main-thread-only (the renderer subscribes to Editor changes and sets `needsDisplay = true`). The first POST `/pointer` crashed the running app until the server was changed to `DispatchQueue.main.async` before invoking the router. A small perf hit on a dev-only server in exchange for not having to put threading guards in every handler.
